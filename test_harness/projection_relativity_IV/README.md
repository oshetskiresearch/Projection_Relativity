# Projection Relativity IV Reproduction Harness

This is the public, paper-supporting PR-IV package. It contains the exact
symbolic certificates, numerical table generators, manuscript coverage audit,
and checked outputs needed to reproduce the published results.

## Reproduce everything

From the repository root, create an isolated Python environment and run:

```powershell
python -m venv .venv-pr4
.venv-pr4\Scripts\python -m pip install -r test_harness\projection_relativity_IV\requirements.txt
$env:OPENBLAS_NUM_THREADS="1"
$env:MKL_NUM_THREADS="1"
$env:OMP_NUM_THREADS="1"
.venv-pr4\Scripts\python test_harness\projection_relativity_IV\symbolic\code\run_all_pr4_audits.py
```

On macOS or Linux, use `.venv-pr4/bin/python` and `export` for the three
thread variables. The master runner exits nonzero if any component fails.

## What the package checks

- 34 dependency-free exact certificates for the seven PR-IV selector
  coordinates and their closure conditions;
- the PR-I radial/compact eigensystem and convergence ledger;
- finite compact, generation, monomial, C2, and D3 scope controls;
- the bounded diagnostic propagation and 103-row branch-control matrix;
- the neutral D3 interval and representative mass table;
- every top-level display and theorem-like claim in the public manuscript;
- unresolved references, citations, and case-sensitive `sections/` paths.

The exact layer uses zero tolerance. Floating-point eigensolver tail digits can
vary with the BLAS implementation, so numerical comparisons use the explicit
absolute tolerances in `numerical_tolerances.json`. The manuscript values are
never used to generate candidates; they are applied only after generation.

## Scope semantics

The theorem is a singleton in the completed admissible class. Broader
P1--P7/common-reduct alternatives are retained as reproducible negative
controls. A control that passes a diagnostic is not counted as an admissible
PR-IV solution because it omits one or more declared canonical-closure rules.
